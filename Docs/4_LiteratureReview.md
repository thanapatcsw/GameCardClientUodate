# 4. ทบทวนวรรณกรรม (Literature Review)

ในการออกแบบและพัฒนาระบบเกมการ์ดเสริมสร้างแนวคิดระบบสารสนเทศ ผู้จัดทำได้ดำเนินการศึกษาและรวบรวมทฤษฎี หลักการ และเทคโนโลยีที่เกี่ยวข้องเพื่อนำมาประยุกต์ใช้เป็นแนวทางในวิจัยและการพัฒนาซอฟต์แวร์จำลอง โดยสามารถสรุปรายละเอียดโครงสร้างแนวคิดได้ดังต่อไปนี้:

- 4.1 ศึกษาระบบงานที่ใกล้เคียง
  - ผู้จัดทำได้ศึกษาและวิเคราะห์สถาปัตยกรรมการทำงาน ข้อดี และข้อจำกัดของแอปพลิเคชันระบบเกมเชิงพาณิชย์และระบบสื่อการเรียนรู้สองระบบเพื่อนำมาใช้เป็นแนวทางในการออกแบบ ได้แก่:
  - 4.1.1 แอปพลิเคชัน Ephod (Mobile Board Game)
    - แอปพลิเคชัน Ephod เป็นเกมจำลองบอร์ดเกม Splendor (เกมค้าเพชร) บนอุปกรณ์เคลื่อนที่ในรูปแบบดิจิทัล โดยเน้นตรรกะการบริหารจัดการทรัพยากร (Resource Management) และการคิดคำนวณแต้มส่วนลดหย่อนถาวร (Discount Logic) เพื่อลดหย่อนราคาการ์ดในเทิร์นถัดไป
    - แนวคิดที่ประยุกต์ใช้: ผู้จัดทำได้นำกลไกการถ่วงดุลทรัพยากร (Game Balancing) และแนวทางการแลกเปลี่ยนการ์ดของ Ephod มาใช้เป็นต้นแบบในการคำนวณสมดุลโครงสร้างของการ์ด 3 ระดับ และการแปลงมูลค่าโบนัสให้เป็นสัญลักษณ์ทรัพยากรระบบสารสนเทศ
    - ข้อจำกัดของระบบเดิม: Ephod มุ่งเน้นการเล่นเพื่อความบันเทิงและไม่ได้สอดแทรกเนื้อหาเชิงวิชาการหรือการประเมินความรู้ที่เป็นระบบ
  - 4.1.2 แอปพลิเคชัน Plato (Social Gaming Platform)
    - Plato เป็นแพลตฟอร์มโซเชียลเกมมิ่งบนอุปกรณ์เคลื่อนที่ที่ได้รับความนิยมสูง จุดเด่นคือระบบการเชื่อมต่อแบบเรียลไทม์หลายผู้เล่น (Real-time Multiplayer) และความเร็วในการตอบสนองซิงโครไนซ์ข้อมูลเครือข่ายระหว่างห้องเล่นที่เสถียร รวมถึงมีฟังก์ชันการสลับสกินกรอบรูปโปรไฟล์และการปรับแต่งตัวละครเพื่อกระตุ้นปฏิสัมพันธ์ของผู้ใช้งาน
    - แนวคิดที่ประยุกต์ใช้: ผู้จัดทำนำรูปแบบการจัดสัดส่วนห้องพัก (Lobby System) การแสดงสถานะผู้เล่นแบบเรียลไทม์ และระบบร้านค้าแลกซื้อสกินโปรไฟล์ผู้ใช้งานของ Plato มาออกแบบโครงสร้างหน้าล็อบบี้และการควบคุมร้านค้าในเกม
    - ข้อจำกัดของระบบเดิม: โครงสร้างสตรีมมิ่งเครือข่ายของ Plato ต้องพึ่งพาการทำงานของเครื่องแม่ข่ายหลักขนาดใหญ่ที่มีค่าใช้จ่ายสูงและมีความซับซ้อนในระดับโปรโตคอล

- 4.2 เครื่องมือและเทคโนโลยีที่ใช้ในการพัฒนาระบบ
  - ในการติดตั้งและพัฒนาแอปพลิเคชันระบบเกมและฐานข้อมูลหลังบ้านจริง ผู้จัดทำเลือกใช้ชุดเทคโนโลยีชั้นนำ (Tech Stack) ที่รองรับการทำงานแบบข้ามแพลตฟอร์มและคลาวด์คอมพิวติ้ง ดังต่อไปนี้:
  - 4.2.1 Unity Game Engine
    - เป็นชุดเครื่องมือพัฒนาซอฟต์แวร์ระบบเกมสากล (Game Engine) รองรับการเขียนสคริปต์ควบคุม Logic ด้วยภาษา C# ทำหน้าที่รันองค์ประกอบด้านกราฟิกจำลอง หน้าจอส่วนติดต่อผู้ใช้งาน (UI) แอนิเมชัน และการตอบสนองเชิงฟิสิกส์ทั้งหมดภายในแอปพลิเคชันเกม ทั้งในฉากการเล่นหลัก (SampleScene.unity) ควิซประจำวัน (DailyQuizScene.unity) และร้านค้า (StoreScene.unity)
  - 4.2.2 Photon Fusion (Shared Mode)
    - เป็นชุดพัฒนาโปรโตคอลเครือข่าย (Network SDK) ประสิทธิภาพสูงที่ทำงานอยู่บนสถาปัตยกรรมคลาวด์จำลอง (Cloud Session) ใช้สำหรับซิงโครไนซ์ข้อมูลสถานะกระดานเกม ลำดับผู้เล่น ข้อมูลทรัพยากรในมือ และข้อมูลคำตอบควิซระหว่างไคลเอนต์แบบเรียลไทม์ในห้องออนไลน์ (Multiplayer Room) ช่วยแก้ปัญหาการเหลื่อมเวลาการตอบและการจัดอันดับความเร็วได้อย่างแม่นยำ
  - 4.2.3 Supabase และระบบฐานข้อมูล PostgreSQL
    - บริการฐานข้อมูลคลาวด์หลังบ้าน (Backend-as-a-Service: BaaS) ที่ใช้ PostgreSQL เป็นระบบจัดการฐานข้อมูล (RDBMS) หลักในการเก็บข้อมูล
    - การเชื่อมโยงระบบ: ตัวเกมเชื่อมต่อ Supabase ผ่าน Gotrue API สำหรับสมัครสมาชิกและยืนยันสิทธิ์ผู้ใช้ (Supabase Auth) และบูรณาการ Postgrest C# Wrapper สำหรับเขียนและอ่านข้อมูลโปรไฟล์ตาราง player_profiles จัดการคะแนนสถิติ MMR และ MMR Leaderboard บนหน้าเกมจริงอย่างมีเสถียรภาพ
    - การประยุกต์ใช้ Database Function: ใช้ฟังก์ชัน SQL ภายในฐานข้อมูล ได้แก่ claim_matchmaking_room และ cancel_matchmaking_search ร่วมกับกลไก advisory lock ในการบริหารจัดคิวในตาราง matchmaking_queue และจัดสรรห้องในตาราง rooms ในระดับเซิร์ฟเวอร์แบบเรียลไทม์เพื่อป้องกันสภาวะแข่งขัน (Race Conditions)
  - 4.2.4 Visual Studio Code และ Visual Studio
    - เครื่องมือแก้ไขและเขียนชุดคำสั่งหลัก (Source Code Editor) ที่ใช้สำหรับการพัฒนาไฟล์ควบคุมภาษา C# ภายในโครงการ Unity และใช้สำหรับพัฒนาสถาปัตยกรรมหน้าเว็บลงทะเบียน (index.html) ที่เชื่อมโยงเข้ากับระบบสมัครสมาชิกของ Supabase Client SDK v2
  - 4.2.5 Android Studio
    - เครื่องมือและสภาพแวดล้อมหลักในการพัฒนาแอปพลิเคชันบนระบบปฏิบัติการ Android ใช้ในขั้นตอนการจำลองฮาร์ดแวร์อุปกรณ์ (Android Emulator) สำหรับทดสอบการแสดงผลและดีพลอยด์สร้างไฟล์ชุดติดตั้ง (.apk) เพื่อทดสอบการติดตั้งใช้งานบนอุปกรณ์สมาร์ทโฟนระบบแอนดรอยด์จริง
  - 4.2.6 Git และ GitHub
    - ระบบควบคุมการจัดเก็บเวอร์ชันของโค้ดโปรแกรม (Version Control System) และพื้นที่จัดเก็บซอร์สโค้ดส่วนกลางบนคลาวด์ ใช้ในการสำรองข้อมูล ติดตามการแก้ไข และประสานงานการพัฒนาไฟล์ร่วมกันระหว่างทีมผู้พัฒนาอย่างเป็นระบบ
  - 4.2.7 Figma
    - แพลตฟอร์มการออกแบบอินเทอร์เฟซออนไลน์ ใช้ในการจัดทำต้นแบบการจัดวางหน้าจอดิจิทัล (UX/UI Wireframes) ออกแบบรูปแบบการ์ดโมดูลซอฟต์แวร์ กรอบรูปโปรไฟล์ และโครงสร้างสี Harmonious Colors ก่อนการสร้างทรัพยากรกราฟิกและอิมพอร์ตเข้าไปใน Unity ในรูปแบบ Prefab
  - 4.2.8 SweetAlert2 (Web Registration UI helper)
    - เป็นไลบรารีสคริปต์สำหรับตกแต่งหน้าต่างแจ้งเตือนและกล่องป๊อปอัปให้มีความทันสมัย สวยงาม และตอบสนองต่อระบบโต้ตอบลงทะเบียนบนหน้าเว็บลงทะเบียน (index.html) โดยช่วยเพิ่มประสิทธิภาพด้านการส่งผ่านข้อมูลแจ้งเตือน (Notifications) ให้เข้าใจง่ายและสะดุดตา
